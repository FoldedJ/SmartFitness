# 小绿书--基于Spring boot + Vue + 达梦数据库DM8的智能健身管理系统
  - 作者：Folded_J
  - 这是本人作为初学者在大三下学期应用软件架构课程的课程设计，相对简陋
    
## 一、项目概述
本项目是一个集成了社交互动、饮食管理与AI智能推荐的综合性健身管理平台。
采用前后端分离架构，前端基于 Vue.js 2 + Element UI 构建，后端采用 Spring Boot 框架，数据库选用国产达梦数据库 (DM8)。
系统不仅提供了基础的用户管理、资讯发布与浏览功能，还深度整合了腾讯混元大模型与扣子（Coze）智能体，为用户提供个性化的饮食建议与营养分析，旨在帮助用户科学管理健康。

**技术栈：**
- **前端：** Vue 2, Element UI, ECharts, WangEditor, Axios
- **后端：** Spring Boot 2.2.4, MyBatis, JWT, Swagger2
- **数据库：** 达梦数据库 DM8
- **AI集成：** 腾讯混元 API, Coze API

## 二、项目配置与部署步骤

### 1. 环境准备
- **JDK:** 1.8
- **Node.js:** 12+ (建议配合 npm/yarn)
- **数据库:** 达梦数据库 DM8 (需开启兼容模式或注意 SQL 语法)
- **开发工具:** IntelliJ IDEA (后端), VS Code (前端)

### 2. 数据库配置
1. 安装并启动达梦数据库服务。
2. 创建相关数据库表结构（请参考 `SQL文件\definition.sql` 脚本，或根据实体类创建表）。
3. 注意：由于 DM8 对关键字（如 USER）敏感，查询时需加双引号，如 `SELECT * FROM SYSDBA."USER"`。

### 3. 后端部署 (smart-fitness-api)
1. 使用 IDEA 打开 `smart-fitness-api` 目录。
2. 修改 `src/main/resources/application.yml` 中的数据库配置：
   ```yaml
   spring:
     datasource:
       url: jdbc:dm://localhost:5236
       username: SYSDBA # 您的数据库用户名
       password: YourPassword # 您的数据库密码
   ```
3. (可选) 配置腾讯混元 API 与 Coze API 密钥（在 `application.yml` 中 `tencent.hunyuan` 等部分）。
4. 运行 `SmartFitnessApplication.java` 启动服务，默认端口 `21090`。
5. Swagger 文档地址: `http://localhost:21090/api/smart-fitness/v1.0/swagger-ui.html`

### 4. 前端部署 (smart-fitness-view)
1. 在终端进入 `smart-fitness-view` 目录。
2. 安装依赖：
   ```bash
   npm install
   ```
3. 启动开发服务器：
   ```bash
   npm run serve
   ```
4. 访问地址: `http://localhost:8080` (默认)。

## 三、功能模块

### 1. 用户模块
- **注册/登录：** 支持用户注册与安全登录 (JWT 认证)。
- **个人中心：** 个人信息管理、身体健康数据录入（身高、体重、性别、出生日期等）。
- **健康档案：** 可视化展示用户的健康数据趋势。

### 2. 社交资讯模块 ("小绿书")
- **资讯浏览：** 首页推荐热门帖子，支持浏览量统计。
- **发布帖子：** 用户可发布图文资讯，分享健身心得。
- **互动：** 支持收藏、点赞、评论（含二级评论）。
- **我的帖子/收藏：** 方便用户管理个人发布与收藏的内容。

### 3. 饮食与营养模块
- **食物库：** 提供常见食物的营养成分查询与搜索，支持用户自定义添加食物。
- **饮食记录：** 用户可记录每日三餐，系统自动计算摄入热量及营养占比。
- **AI 营养建议：** 基于腾讯混元大模型，根据用户的身体指标和饮食记录，提供每日营养摄入推荐。

### 4. 管理员模块
- **用户管理：** 管理注册用户状态。
- **内容管理：** 审核与管理用户发布的帖子、评论。
- **食物管理：** 维护系统基础食物库数据。
- **系统监控：** 查看系统运行状态与统计数据。

## 四、注意事项
### 1.达梦数据库中USER表无法直接访问，需要用双引号包含，如SELECT * FROM SYSDBA."USER" , 也许在安装时通过不同的选项配置可以避免这个问题，但如果你在DM管理工具中浏览USER表数据却提示错误时，尝试以上SQL语句可以解决这个问题。
### 2.配置了腾讯混元api之后，application.yml由于包含敏感信息，没有更新。
### 3.对于Coze相关的配置直接写入了本地环境变量之中。
